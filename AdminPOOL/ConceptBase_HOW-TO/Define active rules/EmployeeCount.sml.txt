*
* Simplified example for Counters with ECArules.
* Note that a single transaction can tell several employees, updating the count attribute.
* We thus have to evaluate (Employee attribute/count i) against the new database state,
* indicated by the back-apostrophe "`" before the condition predicate.
* The Retell action is a combination of Untell (deletion) and Tell (insertion).
}



Class Employee with
  attribute
    count: 0
end


ECArule EmployeeCounterRule_ins with
mode m: Deferred
ecarule
        er : $  x/Employee i,i1/Integer
        ON Tell (x in Employee)
        IF `(Employee attribute/count i) and (i1 = i+1)
        DO Retell (Employee attribute/count i1)
        $
end


ECArule EmployeeCounterRule_del with
mode m: Deferred
ecarule
        er : $  x/Employee i,i1/Integer
        ON Untell (x in Employee)
        IF `(Employee attribute/count i) and (i1 = i-1)
        DO Retell (Employee attribute/count i1)
        $
end

 * to see the updated count attribute.

mary in Employee end
