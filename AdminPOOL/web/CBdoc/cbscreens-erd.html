Screenshots on ERD modeling

Note: This page has not yet been updated for CB 7.0!

This screen shows the ConceptBase workbench with a textual representation of the metaclass 'RelationshipType'. It is part of an example showing how Telos can be used to represent other modeling languages (here: Entity-Relationship diagrams). The constraint captures the semantics of 1:n roles. Note that the constraint quantifies of instances of instances of the metaclass RelationshipType. The lower half shows the protocol window and a window displaying the instances of a metametaclass called Concept.


This is a ConceptBase graph browser specialized for the ERD example. The graphical representations are assigned by user-defined rules. Note that the graph shows three levels of abstraction: the top (language level) is an excerpt of the ConceptBase model for ERD. The middle is an example ER diagram (for some university example), and the bottom shows a set of instances (actual entities and relationships). It is quite easy to change the graphical representation or to include more concepts to the language level.
