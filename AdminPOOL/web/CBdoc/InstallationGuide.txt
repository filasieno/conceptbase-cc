1. System Requirements:
=======================

The deductive object base manager ConceptBase is available for the following six platforms:
 - Solaris 8 or higher on Sparc-compliant machine,
 - Solaris 8 or higher on i386 (Intel CPU)
 - Linux on i386
 - Linux on AMD64/EMT64 (64bit CPU from AMD or Intel)
 - Microsoft Windows NT/2000/XP/Vista on i386 (AMD64 might also work in compatibility mode)

To use the graphical interface you need a Java Runtime Environment compliant to Java V1.4.
The successor version Java 1.5 should also work. Java 6 is unfortunately not 100% compatible 
with one component of the ConceptBase user interface.
The Java runtime environment is available at http://java.sun.com/. 

The hardware requirements are quite low: 32MB main memory and 35MB hard disk space.
Large databases require more main memory and hard disk space. See section 7 for 
more details.

Specific requirements per platform:

Linux/Solaris
 - libstdc++6  (usually part of the gcc 4.x package)

3. Installation:
================

3.1 Using the CBinstaller Program
---------------------------------

To run the CBinstaller program, you must have the Java Runtime
Environment Version 1.4 or newer. 

Download the CBinstaller.jar and the ZIP archive from the ConceptBase
download page. The TAR.GZ archive can not be used by the CBinstaller.
Start the CBinstaller either by a double click on the file CBinstaller.jar,
or by typing

  java -jar CBinstaller.jar

in a command/shell window.

A window should pop up and ask you for the location of the ConceptBase
ZIP archive (cb*.zip) and the destination directory.  Select the ZIP
archive to be extracted and provide an destination directory. 
You can also un-select variants that you do not want to install. We
recommend to leave all variants selected.

Windows: We recommend to use destination directories without blanks or other special
characters in their path. For example, you should not install into "C:\Program Files".
Such pathes can cause problems with the ConceptBase scripts.

By clicking on the "Extract" button, the files will be extracted from
the archive and installed into the specified directory.  Furthermore,
the environment variable CB_HOME will be set to the installation
directory in the script or batch files CBserver[.bat] and CBjavaInterface[.bat].
If the CBinstaller program is running on Windows, it will only adjust
the setting of CB_HOME in the batch files, not in the shell scripts as the
corresponding destination directory on the Unix platform is not known.
The same holds vice versa, if the CBinstaller is running on a Unix platform.

Subsequently, we refer by <CB_HOME> to the absolute path of the directory into
which you installed ConceptBase. Avoid directory names with blanks in the path! 
This can cause problems when starting the ConceptBase user interface. 


3.2 Manual Installation 
-----------------------

If for some reason the CBinstaller method fails or if you prefer manual installation.
then follow these steps:

a. Unzip the file cbXX.zip into a new directory (e.g. /opt/conceptbase or
   c:\conceptbase)

b. Edit in the install directory the files bin/CBserver[.bat], bin/CBshell[.bat]
   and bin/CBshell.bat by replacing the value of the variable CB_HOME with
   your destination directory, i.e. the directory into which you unpacked
   the zip file.

c. Linux/Solaris: make sure that the files in <CB_HOME> are executable,
   e.g. by
     chmod -R ugo+x .


3.3 Upgrading an earlier ConceptBase installation to 7.2
--------------------------------------------------------

If you already have installed ConceptBase 6.1 or 7.0 or 7.1, then you can easily upgrade it
to version 7.2.  Just follow the procedure in section 3.1 and specify the path of 
the existing ConceptBase installation as the destination directory. 
Make sure that ConceptBase is not running before you start the upgrade!


5. Starting ConceptBase:
========================

ConceptBase uses a client-server architecture, i.e. the user interface CBiva and the
server CBserver are separate programs that can also run on different computers.
In fact, if you use ConceptBase in a multi-user setting, then you will want to
start the CBserver on a separate computer, ideally a powerful server with a lot
of main memory. The client CBiva does not require a lot of computing resources and
can run on a standard desktop machine.

There are several ways to start CBiva and CBserver. Method 1 uses the standard scripts
providing all options for setting parameters. This is the preferred way for
regular work with the system. Method 2 is a "quick" way to start the programs with
default settings for the CBserver. This method is suitable if you have installed
ConceptBase on a mobile medium (e.g. a memory stick) and want to show the system
by plugging the mobile medium into an arbitrary PC without installing it there. 
Finally, method 3 is a variant of method 1 by starting the CBserver from within 
the user interface CBiva. This will start the CBserver on the same computer on 
which CBiva is running.


5.1 Method 1 - Standard Way of Starting ConceptBase
===================================================


5.1.1 Starting the ConceptBase Server
-------------------------------------

Open a shell/command window and start the server under Solaris/Linux/ using

    <CB_HOME>/bin/CBserver -p 4001 -d MYDB

or on Windows by using the command

    <CB_HOME>\bin\CBserver.bat -p 4001 -d MYDB


The number 4001 is the port (or "socket") number that is used to connect
 clients to the server process. You can specify any number between 2000 and
65535. The default value is 4001 (if you omit the '-p' option). The parameter
-d specifies the database (sometimes called "application" in the user manual)
to be loaded by the server at startup time. If a path is omitted (as above)
then the database is loaded from the current directory. If there is no
database with the specified name (here:MYDB) then it is created and initialized
with the system classes of O-Telos, the object model of ConceptBase.
After successfully loading the server reports

          This is ConceptBase (CBserver) ...

with some additional information.

Please note that no two servers can use the same port number on the same
machine. In this case you get the error message

	IPC Error: Unable to bind socket to name

and the server won't start up. You should then either start the server on a
different machine or use another port number.

The list of other command line parameters can be obtained by
typing CBserver -h. Specified parameters must be separated by blanks.
See also the ConceptBase User Manual in <CB_HOME>/doc/UserManual for the 
full specification of command line parameters.

You can redirect the output of the CBserver to a log file by a command like
    <CB_HOME>/bin/CBserver -p 4001 -d MYDB  > logfile1.txt
The logfile contains textual output of the CBserver indicating when which
operation was called. The amount of output can be controlled by the
CBserver parameter -t (tracemode). 

Another interesting feature is the parameter -r which directs the CBserver to
restart in case of a crash (again: the unlikely that is). For example,

  <CB_HOME>/bin/CBserver -r 10 -p 4001 -d MYDB  > logfile1.txt

will restart the CBserver 10 seconds after it has crashed. We are not aware of 
interactions that crash ConceptBase 7.0 but the restart options allows to keep
the service stable. The parameter -r is not supported for Windows due to its
limited scripting language.

The CBserver checks the presence of a license key at startup time. If there is
no license key or the license key is expired, it will terminate with a suitable
message. You can easily renew an expired license by contacting the ConceptBase Team
(see end of this file for contact addresses). Some licenses restrict certain
features such as multi-user capability and persistency of updates. In such
cases the settings of the license key override command line parameters.



5.1.2 Starting the ConceptBase Usage Environment CBiva
------------------------------------------------------

The Java-based graphical user interface CBiva can be started on Solaris/Linux by

    <CB_HOME>/bin/CBjavaInterface

or on Windows by

    <CB_HOME>\bin\CBjavaInterface.bat


If starting the application was successful, the main window of CBiva
should appear.

Note, that you must have a Java Runtime Environment, version 1.4
or 1.5. You might have to change the script (CBjavaInterface[.bat])
to start the correct Java Virtual Machine. 


5.1.3 Connecting CBserver and CBiva
-----------------------------------

After having started CBserver and CBiva, the next step is to
establish the connection between them. This is done by selecting menu item "Connect"
from the File menu of CBiva. In the appearing interaction window the
host name and port number of the CBserver must be specified and as a result
the connection is established.

Example:
           1. 	start server on machine alpha with port number 5000 (see above)
           2. 	start usage environment on machine beta (see above)
           3. 	connect usage environment on beta to the server on alpha
              	by selecting Connect CB server and specifying
	  		hostname   : alpha
	  		portnumber : 5000


Whenever a new connection from a usage environment to a CBserver is created a
possibly existing 'old' one is closed. Please note that the machines
alpha and beta can be located anywhere in the Internet. You can also start
both processes on the same machine (use 'localhost' as hostname). 
To connect through a firewall, we recommend the use of secure shell tunnels.


5.2 Method 2 - Starting ConceptBase from a Mobile Medium
========================================================

Once you have installed ConceptBase on your local computer, you can
copy the whole directory of your ConceptBase installation, 
i.e. the directory CB_HOME, on a mobile medium, e.g. a memory stick 
or a CD. It allows you running the system on almost any computer that 
fulfills the minimum system requirements listed in section 1. 

To use ConceptBase on the foreign computer you do not need to install
it there! Nor do you need to copy any file. You just need to switch to
the location where the mobile medium is mounted and start the
system by the scripts
   startCBserver.bat (for Windows)
   startCBserver.sh (for Linux, Unix, Mac)
and
   startCBiva.bat (for Windows)
   startCBiva.sh (for Linux, Unix, Mac)

Under Windows, you can do so by double-clicking the suitable batch files.
Under Linux/Unix/Mac you should type the commands in a shell window.

The script startCBserver starts the CBserver in non-persistent mode with
all other parameters set to their defaults. You need to connect CBiva
to the CBserver just like explained in section 5.1.3. If both scripts
are started on the same computer, you can use 'localhost' as hostname.
The portnumber is to be set to 4001.



5.3 Method 3 - Starting ConceptBase via CBiva
=============================================

If you run the user interface and the CBserver on the same computer, e.g. you
desktop PC, then you can simplify the start procedure by just starting 
CBiva (either with method 1 or method 2) and then starting the CBserver 
via CBiva using the menu option File/Start CBserver. A pop-up window allows
you to change the options of the CBserver. This way is suitable in single-user
settings. If you start the CBserver in this way, you do not need to connect
CBiva to the CBserver. They are automatically connected. See the ConceptBase 
user manual for more details.

Method 3 is convenient but less flexible than method 1. For example, you cannot 
redirect to output of the CBserver to some log file and a crash of CBiva (the unlikely
that may be) also affects the CBserver.



6. Important additional remarks:
================================

A CBserver may only be stopped by a client if the name of the user of the
client is the same as the one who started the CBserver. This behavior is
not an error but a security measure.

We hope the system is useful for your application. We recommend to start with
the ConceptBase tutorials (in <CB_HOME>/doc/Tutorial) for getting used to the
language and the system.

A bibliography on the system is available via the following URL:
http://www-i5.informatik.rwth-aachen.de/CBdoc/cblit.html. 

ConceptBase users are invited to join the ConceptBase Forum. This shared workspace
provides additional documentation, software, and many examples.
More details are available at
  http://www-i5.informatik.rwth-aachen.de/CBdoc/



7. Memory Consumption
=====================

An atomic statement called P-fact in ConceptBase consumes about 1000 bytes
of memory. A typical concept is described by about 10 P-facts (2-4 attributes).
Hence, 1GB can hold around 100.000 concepts. On top of the memory for P-fact,
ConceptBase needs some memory for intermediate results of queries. So, with
2GB you should be able to handle models with about 100.000 concepts.

That may sound low in comparison to database systems but keep in mind that 
ConceptBase is not for storing data but for storing concepts that are part of models.
For example, a database schema with 1000 different classes is regarded as quite
large. In ConceptBase, you can handle schemas that are a hundred times larger.
Our largest models that we encountered had around 10.000 concepts. So, well
in the reach of installed memory of a typical desktop PC.

If your CPU supports 64 bit addresses, there is virtually no limitation
besides the speed of the CPU to process the concepts.
