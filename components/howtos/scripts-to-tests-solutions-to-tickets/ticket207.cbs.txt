# This script demonstrates that the A_e predicate can be used to realize 
# transitivity and symmetry more efficienty. The version *1 is for the
# demonstrating the performance without A_e, the version *2 is for
# demonstrating the performance with A_e. We use cache mode transient here
# in order to prevent any re-use of the cache between two query calls.
# That could distort response times.
#
# The real benefit is gained when coding the meta formulas for symmetry and
# transitivity with A_e. Then, all usages of the attribute categories 'symmetric'
# and 'transitive' will directly benefit. See also definitions in LinkSemantics2.sml
# of the CB-Forum, section HOW-TO/Define basic properties of relations.
#

startServer -u nonpersistent -c transient -t low


# The two versions ModelElement1 and ModelElement2 both define the attribute
# linkto to be symmetric (r1) and transitive (r2).

tell "
ModelElement1 in Class with
  attribute
    linkto: ModelElement1
  rule
    r1: $ forall x,y/ModelElement1 (x linkto y) ==> (y linkto x) $;
    r2: $ forall x,y,z/ModelElement1 ((x linkto y) and (y linkto z)) ==> (x linkto z) $
end

ModelElement2 in Class with
  attribute
    linkto: ModelElement2
  rule
    r1: $ forall x,y/ModelElement2 A_e(x,linkto,y) ==> (y linkto x) $;
    r2: $ forall x,y,z/ModelElement2 (A_e(x,linkto,y) and (y linkto z)) ==> (x linkto z) $
end

"

# Data for both versions; both graphs are isomorphic
tell "
m1 in ModelElement1 with
  linkto l: m2
end

m2 in ModelElement1 with
  linkto l: m3
end

m3 in ModelElement1 with
  linkto l: m4
end

m4 in ModelElement1 with
  linkto l: m5
end

m5 in ModelElement1 with
  linkto l: m6
end

m6 in ModelElement1 with
  linkto l: m7
end

m7 in ModelElement1 with
  linkto l: m8
end

m8 in ModelElement1 end


n1 in ModelElement2 with
  linkto l: n2
end

n2 in ModelElement2 with
  linkto l: n3
end

n3 in ModelElement2 with
  linkto l: n4
end

n4 in ModelElement2 with
  linkto l: n5
end

n5 in ModelElement2 with
  linkto l: n6
end

n6 in ModelElement2 with
  linkto l: n7
end

n7 in ModelElement2 with
  linkto l: n8
end

n8 in ModelElement2 end
"

# These two queries operate on the two isomorphic graphs
tell "
Reachable1 in QueryClass isA ModelElement1 with
  constraint c1: $ (m2 linkto this) $ 
end

Reachable2 in QueryClass isA ModelElement2 with
  constraint c1: $ (n2 linkto this) $
end
"

# This one is based only on the A predicate (not A_e); requires around 0.2 sec on a standard CPU from 2008
ask "Reachable1" OBJNAMES  LABEL  Now

# This one use A_e for generation of faster executable code; requires around 0.01 on a standard CPU from 2008
ask "Reachable2" OBJNAMES  LABEL  Now

# So, the use of A_e yields executable code that is at least one order of magnitude faster than before!
# For large graphs, the performance gain is 10^5 (100.000 times faster).
