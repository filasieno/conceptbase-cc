Creation of Bitmap Icons for ConceptBase GraphBrowser
Manfred Jeusfeld
26-Feb-1998

Besides the predefined shapes (ovals, rectangles, etc.),
the ConceptBase graph browser offers the ability to
assign bitmapped icons as graphical types to nodes.
The bitmap files must be in the XPM format, an ASCII
representation for color bitmaps.

An example on how to assign the XPM bitmap files is contained in
  $CB_HOME/examples/CB_TEAM
or even better refer to the file ExampleWithXPM.sml
in this directory (see also instructions at the end of this file).

This text explain how to create the XPM files. Basically,
there are two methods:


A) From binary format to XPM
============================

You need the utilities 'xv' and 'xgrab' for this method.

Step 1: Display the bitmap with 'xv' on
        the X11 display of a UNIX machine

Step 2: Call 'xgrab'
        Select option Stretch Rectangle, Output Bm/XPM3
        Specify to file, e.g. 'xy.xpm'
        Keep left mouse button pressen while selecting the desired
        bitmap rectangle from the X11 display (screen)

Step 3: Modify the file xy.xpm with the utility 'iconedit'
        if necessary. Note that 'iconedit' is only
        available in OpenWindows, Sun's version of X11.

Some versions of 'xv' already support XPM as output
format. Then you are lucky because you can convert
a bitmap file just by using the "save" option of
xv:

Step 1: Load the binary bitmap file, e.g. 'xv xy.gif'

Step 2: Press right mouse button to display the command
        window of 'xv'

Step 3: Select "save", then "XPM" as output format.

You can delete the lines below the first "};" in the
XPM file (see below for an example).



B) Create an XPM file with a text editor
========================================

All you need is an ASCII text editor like 'vi' to create
a new XPM bitmap file. 

XPM uses characters to denotes colors. In the pixels section,
you have to assign the characters to RGB (red breen blue) colors.

Before these definitions, you have to declare the dimensions
of the picture, e.g.
  20 16 2 1
where 20 is the X dimension (width), 16 is the X dimension (height),
2 is the number of colors and 1 is number of characters denoting
one pixel.

The example below creates a little red arrow in the size 20x16 pixels.

/* XPM */
static char *bsp[] = {
/* width height num_colors chars_per_pixel */
"    20    16        2            1",
/* colors */
". c #ffffff",
"# c #cc0000",
/* pixels */
"#...................",
"###.................",
"#####...............",
"#######.............",
"#########...........",
"###########.........",
"#############.......",
"###############.....",
"###############.....",
"#############.......",
"###########.........",
"#########...........",
"#######.............",
"#####...............",
"###.................",
"#..................."
};



C) Create a Telos model with the new graphical types
====================================================

The example Telos model ExampleWithXPM.sml shows how to
declare a so-called graphical type to be used with the
new icon. First, you have to define a palette like
HintPalette, then the graphical types, e.g. ReadArrowGT,
then the rules on how to assign the graphical types
to Telos objects. Note that for any object x there may
only be one graphical type gt for which (x graphtype gt)
holds. 

In the example, the palette HintPalette is built by 
reusing most of the graphical types of the default
palette X11GraphBrowserPalette. Only the graph type
AGtype_1 replaces the old graphical type AGtype (which
is used to display ordinary attributes of objects).

The two new graph types a1 and a2 are just added.

To work with the example, you have to do the following steps:

Step 1: Start CBserver and CBinterface. Connect them and
        load the Telos model ExampleWithXPM.sml.

Step 2: Go to the menue Options/Graph browser/Palette and
        change the palette to HintPalette (otherwise, the
        object would be displayed according to the default
        palette)

Step 3: Start the graph browser for objet Hint, display its
        instances. Display the attributes of these 
        instances.
