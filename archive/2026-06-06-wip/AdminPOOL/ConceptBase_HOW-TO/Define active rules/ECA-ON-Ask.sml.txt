* This example shows how the call of a query can trigger an ECA rule
* that in turn updates the database. The updates will remain persistent.
* The example is not very meaningful, just to show that it works.
* The rollback time must be set to 'Now' (default). Otherwise, the
* ECA rule will not be triggered.
*
* For technical reasons, the query specified in the ON-part may not be a
* so-called BuiltinQueryClass. 
*
* Requires ConceptBase 7.1 released April 2008 or later.
}


ECArule UpdateOnAsk with
  ecarule
    r1: $ 
         ON Ask find_instances[Integer/class]
         IF TRUE
         DO Tell (1 in Token)
        $
end
