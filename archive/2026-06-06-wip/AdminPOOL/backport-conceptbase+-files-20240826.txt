

Backporting ConceptBase+ files to ConceptBase.cc

Manfred Jeusfeld
2024-08-26 (2024-09-12)




The ConceptBase.cc system was forked into ConceptBase+ at Norgald AB under a
propriatary license in February 2024. From February 2024 to August 2024, this
fork was updated by me to support the modern FlafLaf look and feel.
In additiona, many updates were made to the CBserver for performance improvements
and to the CBIva/CBGraph user interface, in particular to support permanently visible
query and module windows in CBIva.

On August 23, 2024, the board of Norgald AB granted me the permission to backport
the source code changes implemented in ConceptBase+ to the open-source repository
ConceptBase.cc under its FreeBSD license.

This file documents how the backport was done.


(1) Determine the source files that need to be backported

I scanned the history of updated in the GitHub repository of ConceptBase+.
Appendix A contains the list of files that were updated.
Some files like the proprietary copyright statements of ConceptBase+ shall
not be backported to ConceptBase.cc

(2) Write the copy script

The copy script is derived from the content of Appendix A. It copies all
updated files. The script is in Appendix B
Before the script is executed, the symbolic links have to be moved back to ConceptBase.cc

ln -s ~/GitLab/conceptbasecc ~/CBPOOL
ln -s ~/GitLab/conceptbasecc/conceptbase ~/conceptbase

The execute the script-


(3) Some smaller changes

The "about" windows in CBIva is reversed to its old state in ConceptBase.cc




(4) Compile the new binaries

cd ~/CBPOOL
CB_Make clean
compileCB
